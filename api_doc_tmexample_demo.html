Authentication & Authorization

The TMexample API uses API keys for authentication. Each request must include a valid API key in the header:

Authorization: Bearer YOUR_API_KEY

Common errors:

  • 401 Unauthorized: Invalid or missing API key.
  • 403 Forbidden: Insufficient permissions.

Rate Limits and Usage Quotas

To prevent abuse, TMexample API enforces rate limits:

  • 100 requests per minute
  • 5000 requests per day

Exceeding these limits results in a 429 Too Many Requests error.
